# 🍷 와인 포도 품종 250 · 모바일 사전

레드 150종 · 화이트 100종 + 소믈리에 용어집 · 프랑스/이탈리아/독일 등급제 · 프랑스 지역별 특징 & Cru 순위를 담은 모바일 웹앱(PWA)입니다.

## 주요 기능
- **검색**: 한글명 · 원문명 · **한글 초성**(예: `ㅋㅂㄹㄴ` → 카베르네…) 으로 검색
- **구조 필터**: 당도·산도·타닌·바디를 각각 *무관 / 낮음 / 중간 / 높음* 으로 중복 선택
  - `모든 조건(AND)` / `하나라도(OR)` 모드 전환 가능
- **마리아주 추천**: 음식을 고르면 어울리는 품종 **Top 3** 추천
- **포도송이 사진**: 각 품종 상세에서 위키피디아 사진을 실시간 로드(없으면 품종 색 일러스트로 대체)
- **오프라인 지원**: 서비스워커로 앱과 데이터 캐싱 → 네트워크 없이도 사전 사용 가능
- **홈 화면 추가(PWA)**: 스마트폰에서 “홈 화면에 추가”하면 앱처럼 실행

## GitHub Pages 로 배포하기 (무료)
1. GitHub에서 새 저장소(repository)를 만듭니다. 예: `wine-250`
2. 이 폴더의 **모든 파일**(index.html, app.js, data.js, style.css, manifest.webmanifest, sw.js, icons/)을 업로드(또는 push)합니다.
3. 저장소 **Settings → Pages** 로 이동합니다.
4. *Build and deployment* → *Source* 를 **Deploy from a branch** 로 두고, Branch를 `main` / 폴더 `/ (root)` 로 지정 후 **Save**.
5. 1~2분 뒤 `https://<사용자이름>.github.io/wine-250/` 주소가 생성됩니다.
6. 스마트폰 브라우저(사파리/크롬)로 그 주소를 열고 **공유 → 홈 화면에 추가** 하면 앱으로 설치됩니다.

> 사진 기능은 사용자 브라우저에서 위키피디아 API를 직접 호출합니다. GitHub Pages는 https라 정상 동작하며, 별도 서버/키가 필요 없습니다.

## 파일 하나로 바로 쓰기
호스팅 없이 바로 써보려면 함께 제공된 **`와인품종사전_단일파일.html`** 한 개만 스마트폰에 넣고 열면 됩니다(온라인일 때 사진 로드).

## 데이터 갱신
`data.js` 안의 `window.WINE_DATA` / `window.WINE_REF` 를 수정하면 내용이 바뀝니다. 품종 추가 시 `no, ko, en, color, tagline, aromas, structure, cho, lv, q, serving, cultivation …` 필드를 동일 형식으로 채워 주세요.

- 포도송이 사진: 각 품종 위키피디아 페이지 대표 이미지(© 각 저작권자, 상세 화면에 출처 표기)
